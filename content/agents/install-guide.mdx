# Agent 安装指南

本文档汇总常见 Coding Agent / CLI Agent 的安装方式，包括：

- Claude Code
- OpenAI Codex
- Gemini CLI
- Kimi Code
- Qwen Code
- OpenClaw

不同 Agent 的安装方式并不完全相同。有些通过 npm 安装，有些通过 curl / irm 执行官方安装脚本，有些提供桌面端、IDE 插件或云端入口。

因此，Git 和 Node.js 不是所有 Agent 的统一前置条件。是否需要安装，取决于具体工具、安装方式和项目技术栈。

## 2. Git 和 Node.js

### 2.1 Git

Git 不是所有 Agent 的强制依赖，但强烈建议安装，尤其是你希望 Agent 帮你：

- 查看代码变更
- 创建分支
- 回滚修改
- 生成 diff
- 提交 commit
- 创建 Pull Request
- 解决 merge conflict

例如 Claude Code 官方说明它可以直接与 Git 工作，包括 stage、写 commit message、建分支和开 PR；Codex 官方也建议在任务前后创建 Git checkpoints，方便回滚修改。

### 2.2 Node.js

Node.js 是否需要取决于工具：

**必须或通常需要 Node.js 的情况：**

- Gemini CLI：官方运行时要求 Node.js 20.0.0+
- Qwen Code 手动 npm 安装：要求 Node.js 20+
- OpenClaw：推荐 Node 24 或 Node 22.14+
- Codex CLI 使用 npm 安装时：需要 npm / Node.js

**不一定需要用户手动安装 Node.js 的情况：**

- Claude Code 使用 Native Install、Homebrew、WinGet
- Codex App 桌面安装
- Kimi Code CLI 使用官方脚本，通过 uv 安装
- OpenClaw 使用官方脚本时，脚本会按需处理 Node
- 云端 Agent 或浏览器入口

### 2.3 Git 安装教程

#### 2.31 Windows 安装 Git

方法一：官网下载

进入 Git 官方下载页面，选择 Windows 版本下载安装包：

https://git-scm.com/download/win

下载完成后，双击安装包。

安装时大多数选项保持默认即可，建议注意以下选项：

Use Git from the command line and also from 3rd-party software

这个选项可以让你在 PowerShell、CMD、VS Code 终端中直接使用 git 命令。

安装完成后，打开 PowerShell 或 CMD，输入：

```bash
git --version
```

如果看到类似输出，说明安装成功：

```bash
git version 2.x.x
```

#### 2.32 macOS 安装 Git

方法一：使用 Homebrew

如果你已经安装 Homebrew，可以直接执行：

```bash
brew install git
```

安装完成后检查版本：

```bash
git --version
```

方法二：使用官方安装包

也可以进入 Git 官方安装页面选择 macOS 版本下载安装：

https://git-scm.com/download/mac

#### 2.33 Linux / WSL 安装 Git

Ubuntu / Debian

```bash
sudo apt update
sudo apt install git -y
```

检查版本：

```bash
git --version
```

Fedora

```bash
sudo dnf install git -y
```

Arch Linux

```bash
sudo pacman -S git
```

### 2.4 Git 初始配置

安装 Git 后，建议先配置用户名和邮箱。Git 官方文档说明，git config 用于设置 Git 的全局或项目级配置。

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

例如：

```bash
git config --global user.name "YuHang Qin"
git config --global user.email "example@gmail.com"
```

建议把默认分支设置为 main：

```bash
git config --global init.defaultBranch main
```

查看当前 Git 配置：

```bash
git config --global --list
```

### 2.5 安装 Node.js

Node.js 官方下载页会提供当前 LTS 版本和最新版。一般开发环境建议安装 LTS 版本，稳定性更好。npm 官方文档也说明，要使用 npm，需要先安装 Node.js 和 npm，可以通过 Node 安装包或 Node 版本管理器安装。

#### 2.51 Windows 安装 Node.js

进入 Node.js 官方下载页面：

https://nodejs.org/en/download

选择：

Windows Installer (.msi)

建议选择：

- LTS

下载后双击安装包，保持默认选项安装即可。

安装完成后，打开 PowerShell 或 CMD，检查：

```bash
node -v
npm -v
```

如果能看到版本号，说明安装成功：

- v24.x.x
- 11.x.x

#### 2.52 macOS 安装 Node.js

方法一：官网下载

进入 Node.js 官方下载页面：

https://nodejs.org/en/download

选择 macOS Installer，下载 LTS 版本并安装。

检查版本：

```bash
node -v
npm -v
```

方法二：使用 Homebrew

```bash
brew install node
```

检查：

```bash
node -v
npm -v
```

#### 2.53 Linux / WSL 安装 Node.js

Linux / WSL 推荐使用 Node 版本管理器，例如 nvm。npm 官方文档也推荐使用 Node version manager 来安装 Node.js 和 npm。

安装 nvm：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

安装完成后，重启终端，或者执行：

```bash
source ~/.bashrc
```

如果你使用 zsh：

```bash
source ~/.zshrc
```

安装 Node.js LTS：

```bash
nvm install --lts
```

使用 LTS 版本：

```bash
nvm use --lts
```

检查版本：

```bash
node -v
npm -v
```

### 2.6 常用验证命令

安装完成后，可以统一检查：

```bash
git --version
node -v
npm -v
```

如果三条命令都能正常输出版本号，说明 Git、Node.js 和 npm 都已经安装成功。

## 3. Claude Code 安装

Claude Code 是 Anthropic 提供的 Agentic Coding 工具，可以读取代码库、编辑文件、运行命令，并集成终端、IDE、桌面端和浏览器等入口。官方文档列出了 Terminal、VS Code、Desktop app、Web、JetBrains 等使用方式。

### 3.1 macOS / Linux / WSL

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

安装完成后进入项目目录：

```bash
cd your-project
claude
```

首次运行会提示登录。

### 3.2 Windows PowerShell

```bash
irm https://claude.ai/install.ps1 | iex
```

### 3.3 Windows CMD

```bash
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

### 3.4 Homebrew

```bash
brew install --cask claude-code
```

更新：

```bash
brew upgrade claude-code
```

### 3.5 WinGet

```bash
winget install Anthropic.ClaudeCode
```

更新：

```bash
winget upgrade Anthropic.ClaudeCode
```

### 3.6 Git 说明

Claude Code 在 Windows 原生环境下推荐安装 Git for Windows，这样 Claude Code 可以使用 Bash 工具；如果没有安装 Git for Windows，则会回退使用 PowerShell。WSL 环境不需要 Git for Windows。

## 4. OpenAI Codex 安装

Codex 有多种使用方式：CLI、桌面 App、IDE 插件和云端 Codex。Codex CLI 是 OpenAI 的本地终端 Coding Agent，可以读取、修改并运行当前目录中的代码。

### 4.1 Codex CLI：npm 安装

```bash
npm i -g @openai/codex
```

运行：

```bash
codex
```

首次运行时，需要使用 ChatGPT 账号或 API Key 登录。

### 4.2 Codex CLI：Homebrew 安装

```bash
brew install codex
```

运行：

```bash
codex
```

### 4.3 Codex App

Codex App 是桌面端，官方支持 macOS 和 Windows。安装后打开应用，使用 ChatGPT 账号或 OpenAI API Key 登录，然后选择项目文件夹开始使用。

适合场景：

- 想要图形化管理多个 Codex 任务
- 想要查看 diff、worktree、任务线程
- 不想只在终端里操作

### 4.4 Codex IDE 插件

Codex 也提供 IDE 插件，官方 Quickstart 中列出 VS Code、Cursor、Windsurf、VS Code Insiders 等入口。安装后打开 Codex 面板并登录，即可在项目目录中使用 Agent 模式读取文件、运行命令和写入修改。

### 4.5 Codex Cloud

也可以直接在浏览器中使用 Codex Cloud。它需要连接 GitHub 仓库并配置环境，任务完成后可以在 diff 视图中审查修改，也可以创建 Pull Request。

## 5. Gemini CLI 安装

Gemini CLI 是 Google 的终端 AI Agent。官方安装文档列出了系统要求、安装方式和运行方式。当前官方推荐规格包括 macOS 15+、Windows 11 24H2+、Ubuntu 20.04+，运行时要求 Node.js 20.0.0+。

### 5.1 npm 安装

```bash
npm install -g @google/gemini-cli
```

运行：

```bash
gemini
```

### 5.2 Homebrew 安装

```bash
brew install gemini-cli
```

运行：

```bash
gemini
```

### 5.3 MacPorts 安装

```bash
sudo port install gemini-cli
```

### 5.4 Anaconda 环境安装

适合受限环境：

```bash
conda create -y -n gemini_env -c conda-forge nodejs
conda activate gemini_env
npm install -g @google/gemini-cli
```

### 5.5 npx 临时运行

如果不想永久安装，可以使用：

```bash
npx @google/gemini-cli
```

官方文档也提到 Gemini CLI 可以通过 Docker / Podman Sandbox 运行，用于更好的隔离。

## 6. Kimi Code 安装

Kimi Code 提供 CLI 和 VS Code 插件两种主要入口。Kimi Code CLI 是运行在终端里的 AI Agent，可以读写代码、执行 Shell 命令、搜索和抓取网页，并在执行过程中自主规划和调整步骤。

### 6.1 安装前准备

需要满足：

- macOS、Linux 或 Windows PowerShell
- Kimi 账号订阅，或可调用的 API Key

官方文档说明，Kimi Code CLI 安装脚本会先安装 uv，然后通过 uv 安装 Kimi Code CLI。

### 6.2 macOS / Linux

```bash
curl -LsSf https://code.kimi.com/install.sh | bash
```

验证安装：

```bash
kimi --version
```

进入项目目录并启动：

```bash
cd your-project
kimi
```

首次启动后输入：

- /login

### 6.3 Windows PowerShell

```powershell
Invoke-RestMethod https://code.kimi.com/install.ps1 | Invoke-Expression
```

验证安装：

```bash
kimi --version
```

启动：

```bash
kimi
```

### 6.4 使用 uv 手动安装

如果已经安装 uv，也可以执行：

```bash
uv tool install --python 3.13 kimi-cli
```

官方文档说明 Kimi Code CLI 支持 Python 3.12–3.14，并推荐 Python 3.13 以获得更好的兼容性。

### 6.5 VS Code 插件

Kimi Code for VS Code 是集成在 VS Code 中的扩展，支持在编辑器中提问、查看代码 diff、确认修改和提交变更。安装前需要 Kimi 订阅或 API Key；安装方式是通过 VS Code Marketplace 安装。

如果安装后没有显示，可以：

```bash
Developer: Reload Window
```

或重启 VS Code。

## 7. Qwen Code 安装

Qwen Code 是 Qwen 提供的终端 AI Coding Agent。官方 Quickstart 要求先准备终端或命令提示符、一个代码项目，以及 Qwen Code 账号。

### 7.1 Linux / macOS 快速安装

```bash
curl -fsSL https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.sh | bash
```

安装后建议重启终端，让环境变量生效。

### 7.2 Windows 快速安装

官方文档中 Windows 方式要求以管理员身份运行 CMD。推荐使用官方 GitHub README 中较稳的 PowerShell 下载方式：

```powershell
powershell -Command "Invoke-WebRequest 'https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.bat' -OutFile (Join-Path $env:TEMP 'install-qwen.bat'); & (Join-Path $env:TEMP 'install-qwen.bat')"
```

也可以使用 CMD 形式：

```bash
curl -fsSL -o %TEMP%\install-qwen.bat https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.bat && %TEMP%\install-qwen.bat
```

### 7.3 手动安装：npm

手动安装前需要 Node.js 20 或更高版本。

```bash
npm install -g @qwen-code/qwen-code@latest
```

### 7.4 手动安装：Homebrew

```bash
brew install qwen-code
```

### 7.5 登录与启动

启动：

```bash
qwen
```

首次使用时会提示登录，也可以在会话中执行：

- /auth

**或者在终端中执行：**

```bash
qwen auth
```

检查认证状态：

```bash
qwen auth status
```

## 8. OpenClaw 安装

OpenClaw 是一个自托管的 AI Agent Gateway，可以连接 WhatsApp、Telegram、Slack、Discord、iMessage、Matrix、Microsoft Teams、Signal、WebChat 等多个聊天入口，并把消息转给本地或远程 Agent。官方文档说明它运行在用户自己的设备或服务器上。

### 8.1 推荐安装脚本

官方推荐使用安装脚本。该脚本会检测操作系统，按需安装 Node，安装 OpenClaw，并启动 onboarding。

macOS / Linux / WSL2：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Windows PowerShell：

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 8.2 跳过 onboarding

macOS / Linux / WSL2：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard
```

Windows PowerShell：

& ([scriptblock]::Create((iwr -useb https://openclaw.ai/install.ps1))) -NoOnboard

### 8.3 系统要求

官方文档写的是：

- 推荐 Node 24
- 或 Node 22.14+
- 安装脚本会自动处理 Node
- 支持 macOS、Linux、Windows
- WSL2 通常比原生 Windows 更稳定
- pnpm 只有从源码构建时才需要

### 8.4 npm / pnpm / bun 安装

如果你自己管理 Node，可以使用：

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

或者：

```bash
pnpm add -g openclaw@latest
pnpm approve-builds -g
openclaw onboard --install-daemon
```

或者：

```bash
bun add -g openclaw@latest
openclaw onboard --install-daemon
```

### 8.5 验证安装

```bash
openclaw --version
openclaw doctor
openclaw gateway status
```

### 8.6 启动控制台

```bash
openclaw dashboard
```
