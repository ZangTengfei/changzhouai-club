# 参与贡献

第一版我们保持轻量，不追求复杂流程，但希望内容足够清晰、真实、可复用。

## 建议遵循的原则

- 优先写真实经验，不写空泛结论
- 标清楚适用场景，避免脱离上下文
- 一篇文档只解决一个核心问题
- 标题尽量让人看一眼就知道内容范围

## 适合贡献的内容

- 活动组织经验
- 分享准备模板
- 工具选型记录
- 项目协作流程
- 成员复盘与案例

## 不必一开始就很完整

很短的高质量记录，也比“以后再写一篇完整的”更有价值。先写出第一版，再慢慢补齐。

## 推荐的最小贡献方式

1. 修正一处表达不清的地方
2. 增加一个参考链接
3. 补一段真实的操作步骤
4. 新增一篇短小但明确的经验记录
