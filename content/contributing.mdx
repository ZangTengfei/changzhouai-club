# 参与贡献

常州 AI Club 文档已经进入正式使用阶段，但共建方式仍然保持轻量。我们更看重内容真实、上下文清楚、能被后来者复用。

## 建议遵循的原则

- 优先写真实经验，不写空泛结论
- 标清楚适用场景，避免脱离上下文
- 一篇文档只解决一个核心问题
- 标题尽量让人看一眼就知道内容范围
- 涉及个人、客户或项目敏感信息时，先确认是否适合公开

## 适合贡献的内容

- 活动记录、活动复盘和主题分享整理
- 从活动记录中拆出来的专题文章
- 项目协作流程、需求梳理方法和交付经验
- 成员实践案例、工具工作流和踩坑记录
- 社区运营、内容发布和成员共建经验

## 推荐的贡献方式

### 修正现有内容

适合第一次参与：

- 修正错字、人名、项目名或时间
- 补充缺失链接
- 调整标题和段落结构
- 给长文补充小节标题

### 补充活动沉淀

适合参加过活动的成员：

- 补充现场观点
- 增加项目背景
- 标注后续值得跟进的主题
- 把一段讨论整理成更清晰的段落

### 新增专题文章

适合已经有明确经验的人：

- 从某次活动记录里拆出一个专题
- 总结一个项目流程
- 复盘一次工具使用或业务实践
- 写一份能被后续成员直接复用的模板

## 推荐的最小贡献方式

1. 修正一处表达不清的地方
2. 增加一个参考链接
3. 补一段真实的操作步骤
4. 新增一篇短小但明确的经验记录

## 内容公开边界

活动记录里可能会出现项目线索、客户场景、成员背景或合作需求。贡献时请注意：

- 不公开未经确认的个人联系方式
- 不公开客户敏感信息、报价细节或合同内容
- 不把群聊里的内容直接搬运成公开文档
- 不确定能否公开时，先保留模糊表达或等待确认

文档的目标是沉淀社区知识，而不是暴露不该公开的信息。
