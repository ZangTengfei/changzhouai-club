# Nextra 与 Mintlify 选型记录

结合当前社区官网的技术栈，我们优先采用 Nextra。

## 当前结论

- 现有官网已经是 Next.js App Router 项目
- 知识库先放进同一个 Git 仓库更利于协作
- 第一阶段更看重可控性、低成本和易维护

## 为什么先选 Nextra

### 技术贴合度更高

当前项目本身就是 Next.js 应用，因此直接在同一应用内新增 `/docs` 路由，集成成本最低。

### 内容和品牌能共用

站点域名、品牌元素、导航关系和部署流程都可以和主站共享，不需要再拆一套系统。

### 更适合 Git 驱动共建

第一版知识库更适合通过 Git + PR 的方式持续共建，而不是一开始就引入独立平台工作流。

## Mintlify 更适合什么场景

如果未来社区知识库变成一个大规模的外部开发者文档站，需要更强的托管能力、产品化后台和团队编辑体验，再考虑 Mintlify 会更合适。
